# Objetivo del Proyecto

> Nota importante
> Este documento esta sujeto a modificaciones conforme evolucione el proyecto.
> Sin embargo, en este momento representa el objetivo oficial que guia al equipo.

## Objetivo actual del proyecto

LogicKids es un proyecto de grado del SENA orientado a apoyar al tutor en la observacion y evaluacion del razonamiento logico de ninos mediante una experiencia digital ludica. El sistema permite registrar tutores y estudiantes, controlar el acceso del nino a las sesiones diagnosticas, capturar evidencia de desempeno durante los minijuegos y presentar resultados utiles para identificar fortalezas, dificultades y necesidades de acompanamiento.

El foco principal del proyecto no es la ensenanza directa, sino la generacion de informacion diagnostica que ayude al tutor a comprender como se desempena cada nino y como proceder con su seguimiento. Para ello, LogicKids integra backend, frontend web, experiencia movil y minijuegos diagnosticos dentro de una misma solucion.

Como producto academico y tecnico, el proyecto se construye con una regla central: la base de datos oficial define la verdad del sistema. A partir de esa verdad se alinean los flujos funcionales, las reglas del backend, las interfaces del frontend y la evolucion futura del producto.

## Proyeccion futura

Como linea de evolucion, y siempre que el tiempo del proyecto lo permita, LogicKids podra ampliarse con nuevas funcionalidades orientadas a enriquecer el analisis diagnostico y la utilidad del sistema para el tutor. Entre estas posibilidades futuras se contemplan la incorporacion de mas minijuegos diagnosticos, la ampliacion de la experiencia movil, la generacion de recomendaciones mas avanzadas e incluso el apoyo de inteligencia artificial para fortalecer la interpretacion de resultados.

Estas proyecciones no cambian el objetivo principal del proyecto, sino que representan posibles extensiones sobre una base ya funcional, coherente y alineada con la base de datos oficial del sistema.

## Regla inquebrantable del proyecto

La DB oficial manda.

Si una historia, una pantalla, una ruta o una decision tecnica contradice la base de datos oficial, debe corregirse la implementacion o la documentacion. No se inventan reglas por fuera de la verdad definida en la DB.
